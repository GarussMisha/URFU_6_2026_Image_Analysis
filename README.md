# IPCAM - Детектор Движения для IP-Камер

## 📋 Описание проекта

**IPCAM** — это инструмент для мониторинга и обнаружения движения с использованием IP-камер, поддерживающих RTSP-поток. Приложение непрерывно анализирует видеопоток в реальном времени и автоматически сохраняет снимки при обнаружении значимого движения.

### Ключевые возможности:

- **Анализ RTSP-потока** — работа с любыми IP-камерами, поддерживающими RTSP-протокол
- **Детекция движения** — алгоритм сравнения кадров для выявления изменений
- **Асинхронное сохранение** — фоновый поток для записи снимков без задержек
- **Гибкая настройка** — множество параметров для тонкой настройки чувствительности
- **Минимальная нагрузка** — оптимизация через пропуск кадров между анализами
- **Автоматическое переподключение** — восстановление соединения при потере связи с камерой
- **Отладочные окна** — визуализация текущего кадра и порога для настройки детекции
- **Статистика** — подсчёт сохранённых снимков и ошибок при завершении

## 🚀 Быстрый старт

### 1. Клонирование репозитория

```bash
git clone <repository-url>
cd IPCAM
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

Или установка основных пакетов:

```bash
pip install opencv-python numpy python-dotenv onvif-zeep
```

### 3. Настройка подключения к камере

Скопируйте шаблон конфигурации и заполните параметры:

```bash
cp .env.example .env
```

Откройте файл `.env` и укажите данные подключения:

```env
RTSP_USERNAME=admin
RTSP_PASSWORD=your_password
RTSP_IP=192.168.1.100
RTSP_PORT=554
RTSP_STREAM=stream1
```

**Параметры подключения:**
| Параметр | Описание |
|----------|----------|
| `RTSP_USERNAME` | Логин для доступа к камере |
| `RTSP_PASSWORD` | Пароль для доступа к камере |
| `RTSP_IP` | IP-адрес камеры |
| `RTSP_PORT` | Порт RTSP (обычно 554) |
| `RTSP_STREAM` | Поток: `stream1` (основной) или `stream2` (вторичный) |

### 4. Запуск приложения

```bash
python motion_detector.py
```

## ⚙️ Настройка параметров

Все параметры детекции находятся в файле [`config.py`](config.py):

### Чувствительность детекции

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `MIN_AREA` | 8000 | Минимальная площадь движения (пиксели). Меньше = чувствительнее |
| `THRESHOLD` | 35 | Порог различия кадров (0-255). Меньше = чувствительнее |
| `MOTION_DURATION` | 10 | Мин. кадров движения перед сохранением |
| `COOLDOWN_SECONDS` | 2 | Время ожидания после снимка |
| `FRAME_SKIP` | 2 | Пропуск кадров между анализами |

### Рекомендации по настройке:

- **Высокая чувствительность:** `MIN_AREA=3000`, `THRESHOLD=20`
- **Низкая чувствительность:** `MIN_AREA=15000`, `THRESHOLD=50`
- **Меньше нагрузки CPU:** увеличьте `FRAME_SKIP`

## 📊 Выходные данные

При обнаружении движения приложение сохраняет снимки в директорию `motion_snapshots/` с именами формата `motion_XXX.jpg`.

### Отладочные окна

Во время работы приложения отображаются два окна:
- **Motion Detection** — текущий кадр с детекцией
- **Threshold** — пороговое изображение для визуальной настройки

### Статистика при завершении

При нажатии `q` или `Ctrl+C` программа выводит статистику:
- Общее количество сохранённых снимков
- Количество ошибок сохранения (если были)

## 🔄 Дополнительные функции

### Автоматическое переподключение
При потере соединения с камерой приложение автоматически пытается переподключиться (до 10 попыток с интервалом 5 секунд).

### Cooldown (остывание)
После сохранения снимка активируется период ожидания (`COOLDOWN_SECONDS`), предотвращающий сохранение множества снимков подряд.

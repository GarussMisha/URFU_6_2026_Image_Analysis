# IPCAM — Детектор движения для IP-камер

## О проекте

**IPCAM** — система автоматической детекции движения для IP-камер, подключённых по протоколу RTSP. Приложение непрерывно анализирует видеопоток в реальном времени и сохраняет снимки при обнаружении значимого движения.

### Как это работает

Детекция основана на алгоритме разницы кадров (frame differencing):

1. Первый кадр используется как «фон» — эталонное изображение без движения.
2. Каждый следующий кадр сравнивается с фоном через `absdiff` — вычисляется абсолютная разница.
3. Пиксели, разница которых превышает порог `THRESHOLD`, считаются изменёнными.
4. Результат размывается (dilate) для объединения разрозненных участков в единый контур.
5. Находятся контуры и вычисляется их суммарная площадь.
6. Если площадь больше `MIN_AREA` — засчитывается один кадр движения.
7. При накоплении достаточного количества кадров движения (`MOTION_DURATION`) — сохраняется снимок.

### Особенности

- **Асинхронное сохранение** — снимки сохраняются в фоновом потоке, не блокируя детекцию.
- **Автоматическое переподключение** — при потере связи с камерой приложение пытается переподключиться до 10 раз.
- **Cooldown** — период ожидания после снимка предотвращает дублирование.
- **Пропуск кадров** — анализ не каждого кадра снижает нагрузку на CPU.
- **Адаптивный фон** — эталонный кадр обновляется после каждого анализа.

## Структура проекта

```
IPCAM/
├── motion_detector.py    # Основной модуль детекции
├── config.py             # Конфигурация параметров
├── .env                  # Параметры подключения к камере
├── .env.example          # Шаблон конфигурации
├── requirements.txt      # Зависимости Python
├── .gitignore            # Исключения Git
└── motion_snapshots/     # Директория для снимков
    └── motion_000.jpg
```

### Основные файлы

| Файл | Описание |
|------|----------|
| `motion_detector.py` | Два класса: `MotionDetector` (детекция движения) и `AsyncSnapshotSaver` (асинхронное сохранение) |
| `config.py` | Параметры подключения (из `.env`) и настройки детекции |
| `.env` | Переменные окружения: логин, пароль, IP, порт, поток камеры |

## Запуск

### 1. Клонирование

```bash
git clone https://github.com/GarussMisha/URFU_6_2026_Image_Analysis.git
cd IPCAM
```

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

Основные зависимости:
- `opencv-python` — детекция движения и работа с камерой
- `numpy` — вычисления
- `python-dotenv` — загрузка `.env`

### 3. Настройка подключения

Скопируйте шаблон и заполните параметры:

```bash
cp .env.example .env
```

```env
RTSP_USERNAME=admin
RTSP_PASSWORD=your_password
RTSP_IP=192.168.1.100
RTSP_PORT=554
RTSP_STREAM=stream1
```

**Параметры подключения:**

| Параметр | Описание |
|----------|----------|
| `RTSP_USERNAME` | Логин для доступа к камере |
| `RTSP_PASSWORD` | Пароль для доступа к камере |
| `RTSP_IP` | IP-адрес камеры |
| `RTSP_PORT` | Порт RTSP (обычно 554) |
| `RTSP_STREAM` | Поток: `stream1` (основной) или `stream2` (вторичный) |

### 4. Запуск

```bash
python motion_detector.py
```

При запуске откроются два окна:
- **Motion Detection** — текущий кадр с камеры.
- **Threshold** — пороговое изображение (белым показаны области движения).

Для выхода нажмите `q`.

## Настройка параметров

Все параметры детекции находятся в `config.py`:

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `MIN_AREA` | 6000 | Минимальная площадь движения (пиксели). Меньше = чувствительнее |
| `THRESHOLD` | 30 | Порог различия кадров (0-255). Меньше = чувствительнее |
| `MOTION_DURATION` | 5 | Мин. кадров движения перед сохранением |
| `COOLDOWN_SECONDS` | 2 | Время ожидания после снимка |
| `FRAME_SKIP` | 2 | Пропуск кадров между анализами |

### Рекомендации

- **Высокая чувствительность:** `MIN_AREA=3000`, `THRESHOLD=20`
- **Низкая чувствительность:** `MIN_AREA=15000`, `THRESHOLD=50`
- **Меньше нагрузки CPU:** увеличьте `FRAME_SKIP`

## Выходные данные

Снимки сохраняются в `motion_snapshots/` с именами формата `motion_XXX.jpg`.

При завершении (нажатие `q` или `Ctrl+C`) выводится статистика:
- Общее количество сохранённых снимков
- Количество ошибок сохранения
